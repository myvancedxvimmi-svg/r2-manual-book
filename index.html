R2PCKGZ1_Dev Bot

Manual book interaktif — Telegram Maintenance Bot untuk laporan kerusakan mesin pabrik (Zona 1, Group R2).

Google Apps Script Telegram Webhook Gemini + Groq AI Spreadsheet DB Dark · Interactive Web App Access

1. Pengenalan

Bot ini menerima laporan kerusakan mesin dari teknisi lewat Telegram, mengekstrak info (Line, Mesin, Masalah) menggunakan AI, menyimpannya ke Google Spreadsheet, lalu mengirim rekap ke thread Hasil Rekapan saat shift selesai.

Masalah yang diselesaikan: Teknisi biasanya mengetik laporan bebas (typo, multi-line, format acak). Bot otomatis merapikan jadi struktur baku tanpa teknisi harus isi form.

Komponen Utama

  • doPost(e) — Webhook handler, titik masuk semua pesan Telegram.
  • doGet() — Serve manual book ini sebagai web app (akses siapa saja).
  • safeGetSheet() — Manajemen tab Spreadsheet (auto-create).
  • formatWithAI() — Ekstrak teks bebas jadi array JSON via AI.
  • askGemini() — Balas pertanyaan bebas di thread "Ask With AI".
  • kirimRekapan() / selesaiShift() — Pipeline rekap akhir shift.

2. Arsitektur Sistem

Alur data dari pesan masuk sampai tersimpan:

Telegram  ──webhook──▶  doPost(e)
                             │
                  ┌──────────┴───────────┐
            Callback Query?          Pesan Teks?
                  │                      │
          (approve/reject)        Thread = Input Pekerjaan?
                                          │
                                  formatWithAI(text)
                                          │  Gemini 2.5-flash
                                          │  → 2.0-flash-lite (fallback)
                                          │  → Groq llama-3.1-8b (fallback)
                                          ▼
                                  Parse JSON [line, mesin, masalah]
                                          │
                                  formalize() → kapital + titik
                                          │
                                  sheet.appendRow → "Data"
                                          │
                                  sendMessage → "✅ Dicatat: ..."
Respon wajib HTTP 200: Webhook mengembalikan HtmlService.createHtmlOutput("OK") di baris pertama sebelum logic apa pun, untuk mencegah Telegram melakukan retry loop (error 302 Found).

5. Daftar Perintah (/)

Perintah diawali / dan diproses di thread manapun. Ketik / di Telegram untuk lihat semua dengan deskripsi.

Fungsi: Tampilkan semua laporan tersimpan di tab Data beserta nomor urutnya.
Cara: /review (tanpa argumen).
Jika kosong → balas "Kosong." Nomor urut dipakai untuk /hapus.
Fungsi: Hapus baris ke-[no] dari review (berdasar nomor di /review).
Cara: /hapus 2 → hapus baris nomor 2 di sheet Data.
Validasi: rowNum > 0 && rowNum < sheet.getLastRow(). Di luar range → abaikan.
Fungsi: Kosongkan seluruh isi tab Data dan Memory (baris setelah header dihapus).
Hati-hati: ini menghapus laporan yang belum dikirim. Tidak menyentuh Archive.
Fungsi: Tampilkan preview semua laporan + inline keyboard ✅ Approve / ❌ Reject.
  • Approve → jalankan kirimRekapan() ke thread Hasil Rekapan, lalu edit pesan jadi "✅ Rekapan dikirim".
  • Reject → edit pesan jadi "❌ Rekapan dibatalkan".
Jika Data kosong → "⚠️ Data kosong, tidak ada rekapan untuk dikirim."
Fungsi: Pipeline akhir shift:
1. kirimRekapan() ke Hasil Rekapan.
2. Pindahkan semua baris Data → Archive (split Mesin / Masalah).
3. Hapus isi Data.
4. Balas "✅ Shift selesai! Data shift ini sudah di-archive ke sheet Archive."
Fungsi: Daftarkan mesin ke master data Inventory (sheet). Dipakai agar bot bisa mapping Line otomatis saat laporan hanya sebut nama mesin.
Cara: /tambahmesin <nama> <line> [kode] [equip] [notif]
Argumen wajib: nama, line. Opsional: kode mesin, no equipment, no notif.
Contoh:
/tambahmesin conveyor 2 → daftar conveyor di Line 2.
/tambahmesin sealer 1 AA 123456 9981 → nama sealer, Line 1, kode AA, equip 123456, notif 9981.
Jika arg < 3 → balas "⚠️ Format salah. Contoh: /tambahmesin conveyor 2 AA 123456 9981"
Fungsi: Isi Kode Mesin di Inventory (khusus mesin ultra yang punya kode). Baris dibuat otomatis jika belum ada.
Cara: /kode <kode> <mesin> <line>
Contoh: /kode AA Mesin 2 1 → simpan kode AA untuk Mesin 2 Line 1.
Kode ini akan muncul di laporan hanya saat /mode formal aktif.
Fungsi: Isi No Equipment mesin di Inventory. Semua mesin punya no equip.
Cara: /equip <no_equip> [mesin] [line]
Line opsional jika mesin unik di 1 line.
Contoh:
/equip 123456 1 → no equip 123456 untuk mesin di Line 1.
/equip 998100 Mesin 2 1 → no equip 998100 untuk Mesin 2 Line 1.
No equip muncul di laporan hanya saat /mode formal aktif.
Fungsi: Atur label & gaya laporan. Simpan persisten di sheet Template.
Label (identity):
• /mode mc → pakai Mc (default, identity tim). Contoh: Line 10 Mc 2: ...
• /mode mesin → pakai Mesin (lebih formal). Contoh: Line 10 Mesin 2: ...
Gaya:
• /mode nonformal → ringkas (default): Line 10 Mc 2: Pergantian belt variable putus. Notif 123456.
• /mode formal → lengkap (Kode + Equip + Notif): Line 10 Mc 2 AA 998100: Pergantian belt variable putus. Notif 123456.
Label & gaya terpisah — bisa /mode mc + /mode formal barengan. Berlaku sampai diubah. Tanpa arg → balas status saat ini.
Cek 5 riwayat perbaikan terakhir mesin dari Data + Archive.
Nama mesin di-match sebagian (partial) ke Inventory untuk dapat Line.
Jika tidak terdaftar → "⚠️ Mesin tidak terdaftar di Inventory..."
Fungsi: Ubah format laporan rekap lewat chat (gak perlu edit script). Tersimpan di Template key REKAP_TEMPLATE.
Penanda wajib: {TANGGAL} (tanggal otomatis), {DAFTAR} (daftar kerusakan per sektor).
Cara: /settemplate <teks penuh>
Contoh:
/settemplate Report Packaging Zona 1\nGroup R2\n{TANGGAL}\n\nMasalah DT >60 menit\n- Sektor 1:\n...\n\nDaftar Pekerjaan\n{DAFTAR}\n\nPengajuan Sparepart:\n\nKecelakaan Kerja:
Kalau PT ganti format laporan → cukup ketik /settemplate baru. Default kembali jika baris dihapus.
Tampilkan format rekap yang sedang aktif (isi REKAP_TEMPLATE).
Jika belum pernah di-set → balas template default bawaan bot.
Tampilkan laporan status API (static report: jumlah key terdaftar, status aktif).
Tampilkan daftar semua perintah di atas (teks ringkas di Telegram).

6. Alur Laporan Kerusakan

Saat teknisi mengetik di thread Input Pekerjaan (bukan perintah):

  1. Webhook terima → doPost(e) → log "PESAN DITERIMA".
  2. Anti-spam & dedup cache cek.
  3. formatWithAI(text) panggil AI → dapat array JSON.
  4. Untuk tiap item: cari Line (jika kosong, map dari Inventory via nama mesin).
  5. Jika Line tetap kosong → tolak dengan pesan warning.
  6. formalize() → kapital huruf pertama + diakhiri titik + auto-correct typo (mc→Mesin, convyer→Conveyor, dll).
  7. Ambil Kode Mesin & No Equip dari Inventory (jika mode formal).
  8. Simpan ke Data: Line X Mc Y: masalah. Notif Z.
  9. Balas ringkas: ✅ Dicatat:\n- Line 14: pergantian belt selaer\n- Line 15: ...
Input : "Line 14 15 16 pergantian belt selaer notif 5521"
AI    : [{"line":"14","mesin":"","masalah":"pergantian belt selaer"},
         {"line":"15",...},{"line":"16",...}]
Simpan: Line 14 Mc -: Pergantian belt selaer. Notif 5521.
        Line 15 Mc -: Pergantian belt selaer. Notif 5521.
        Line 16 Mc -: Pergantian belt selaer. Notif 5521.
Balas : ✅ Dicatat:
        - Line 14 Mc -: Pergantian belt selaer.
        - Line 15 Mc -: Pergantian belt selaer.
        - Line 16 Mc -: Pergantian belt selaer.
No Notif: Nomor order dari produksi (SAP). Diekstrak otomatis dari teks "notif 5521" / "no notif 9981" → disimpan kolom ke-3 & di-append " Notif 5521." di akhir laporan.
Multi-line support: Teks "semua line dari 1 sampe 21 mati lampu" diekstrak jadi 21 object terpisah (Line 1..21). Balasan digabung jadi 1 pesan agar tidak spam Telegram (batas 4000 char dipotong).

9. Engine AI & Fallback

Dua fungsi AI utama, masing-masing punya rantai fallback agar bot tidak bisu saat quota habis:

FungsiUrutan Fallback
formatWithAI()gemini-2.5-flash (7 key) → gemini-2.0-flash-lite (7 key) → Groq llama-3.1-8b (2 key)
askGemini()gemini-2.5-flash → 2.0-flash-lite (chat mode) → Groq llama-3.1-8b

Helper internal: callGemini(prompt, keys, cfg) & callGeminiChat(messages, keys) — loop model × key, return teks pertama yang sukses.

Format output wajib: AI harus balis array JSON murni tanpa markdown. Jika ada ```json dibersihkan otomatis. Parse gagal → return null → bot balas "⚠️ Format salah. Gagal mengekstrak laporan kerusakan."

Vision (Foto)

Jika foto dikirim di thread Ask With AI → analyzePhotoWithGemini() download file, kirim ke Gemini Vision, balas analisa kerusakan.

Quota 429: Gemini free-tier membatasi ~20 req/menit per model. Fallback ke model ringan (2.0-flash-lite) punya pool quota berbeda sehingga respon tetap jalan saat 2.5-flash limit.

12. Cara Akses Manual Book (User Lain)

Manual book ini di-deploy sebagai Web App Google Apps Script, sehingga siapa saja bisa membukanya lewat browser tanpa akses ke source code.

Langkah untuk user lain:
  1. Buka URL Web App yang di-share admin (format: https://script.google.com/macros/s/<DEPLOY_ID>/exec).
  2. Akses ANYONE_ANONYMOUS → tidak perlu login Google.
  3. Gunakan kolom Cari section... di kiri untuk filter cepat, atau klik daftar isi.
Admin: Setiap clasp push && clasp deploy menghasilkan deployment baru. Agar URL stabil, gunakan --deploymentId yang sama saat deploy update, atau share URL versi terakhir ke user. Pastikan doGet() ada di Code.js (sudah ditambahkan).
# Deploy web app (simpan ID ini untuk share ke user)
clasp deploy
# Output: Deployed AKfycby... @NN  ->  URL:
# https://script.google.com/macros/s/AKfycby.../exec

13. Troubleshooting

Cek DebugLogs sheet. Urutan log normal:
1. PESAN DITERIMA: {...}
2. DEBUG: Pesan terdeteksi, mulai proses...
3. DEBUG: Akan mengirim balasan ke Telegram...
4. DEBUG: Selesai.
Jika mandek di step 2 → AI gagal (cek raw response 429). Jika mandek sebelum step 1 → webhook mati.
Pastikan doPost mengembalikan HtmlService.createHtmlOutput("OK") (bukan ContentService yang memicu redirect). Lalu reset webhook via setWebhook.
Gemini free-tier limit ~20 req/menit. Bot otomatis fallback ke gemini-2.0-flash-lite lalu Groq. Jika semua mati → balas "Semua AI key sibuk/limit. Coba lagi nanti." Tunggu ~1 menit.
Dedup cache 5 menit aktif. Jika masih dobel, cek apakah pesan dikirim dari bot lain (bukan R2PCKGZ1_Dev_bot) — hanya bot sendiri yang diabaikan.
Jika AI tidak dapat Line tapi dapat nama mesin, bot cari Line dari Inventory (partial match). Jika mesin juga tidak terdaftar → laporan ditolak dengan warning.
Pastikan sudah /mode formal. Kode Mesin & No Equip hanya ditampilkan di gaya formal. Cek juga data di Inventory sudah diisi via /kode & /equip.

14. Deploy & Webhook

Workflow standar pengembangan:

# Edit Code.js lokal
clasp push          # upload ke Apps Script
clasp deploy        # buat version deployment baru (web app + webhook)
# Reset webhook agar Telegram arahkan ke URL deploy terbaru
curl -X POST "https://api.telegram.org/bot<TOKEN>/setWebhook?url=<URL_DEPLOY_BARU>"

Cek status webhook kapan saja:

curl -X POST "https://api.telegram.org/bot<TOKEN>/getWebhookInfo"
Penting: Setiap clasp deploy menghasilkan URL baru. Webhook harus di-reset ke URL tersebut, jika tidak Telegram masih menembak deployment lama. URL yang sama dipakai untuk akses manual book (doGet).

Debug Logs

Semua log tertulis di sheet DebugLogs (Tidak perlu buka menu Executions yang sering error). Log PESAN DITERIMA dipotong max 200 char agar sheet tidak membengkak.
